# Sliding Window Advertisement

## Problem

- **Source:** [CSES 3227: Sliding Window Advertisement](https://cses.fi/problemset/task/3227/)
- **Code:** [`View accepted C++ solution (advertisement.cpp)`](../advertisement.cpp)
- **Limits:** $1\le k\le n\le2\cdot10^5$ and $1\le a_i\le10^9$.

For every contiguous window of $k$ fence boards, find the maximum area of an axis-aligned rectangle that can be inscribed beneath the board heights.

For a contiguous range $[u,v]$, the largest rectangle using exactly that range has area

$$
(v-u+1)\min_{u\le j\le v}a_j.
$$

## Idea

Calculating the largest histogram rectangle independently for every window costs $O(nk)$ overall, even if the standard linear monotonic-stack algorithm is used for each individual window.

Instead, treat every board height $a_i$ as the height of a candidate rectangle and determine its contribution as the window moves.

Let $[l_i,r_i]$ be the largest interval containing $i$ such that

$$
a_j\ge a_i
\qquad
\text{for every }l_i\le j\le r_i.
$$

The boundaries lie immediately after the nearest strictly smaller board on the left and immediately before the nearest strictly smaller board on the right. Two monotonic-stack passes compute all intervals in $O(n)$ time.

Now consider a window beginning at $s$:

$$
[s,s+k-1].
$$

The longest rectangle of height $a_i$ available in this window is the intersection of the window with $[l_i,r_i]$. Its width is

$$
w_i(s)=\min(r_i,s+k-1)-\max(l_i,s)+1
$$

whenever the intervals overlap.

As $s$ moves from left to right, this width first increases, then remains constant, then decreases. Let

$$
p=\min(l_i,r_i-k+1),
\qquad
q=\max(l_i,r_i-k+1).
$$

Then

$$
w_i(s)=
\begin{cases}
s+k-l_i, & l_i-k+1\le s<p,\\
\min(k,r_i-l_i+1), & p\le s\le q,\\
r_i-s+1, & q<s\le r_i.
\end{cases}
$$

Multiplying by $a_i$ gives three linear area functions:

$$
a_i s+a_i(k-l_i),
$$

$$
a_i\min(k,r_i-l_i+1),
$$

and

$$
-a_i s+a_i(r_i+1).
$$

Only valid window starts $0\le s\le n-k$ are inserted.

The problem is therefore reduced to adding lines over ranges of window starts and finding the maximum at every start position. A segment tree handles the validity ranges, while each node maintains a Li Chao envelope over its coordinate interval.

## Algorithm

1. Use a monotonic stack from left to right to compute $l_i$, the first position after the nearest strictly smaller value to the left.
2. Use a monotonic stack from right to left to compute $r_i$, the last position before the nearest strictly smaller value to the right.
3. For every board $i$:
   - Derive the increasing segment.
   - Derive the constant segment.
   - Derive the decreasing segment.
   - Clip each segment to valid window starts and insert it into the Li Chao segment tree.
4. For every window start $s$ from $0$ through $n-k$, query the maximum stored value.
5. Output the query results.

## Correctness

We prove that every query result is the maximum advertisement area in its window.

**Lemma 1.** For each index $i$, every board in $[l_i,r_i]$ has height at least $a_i$, and this interval cannot be extended while preserving that property.

**Proof.** During the left monotonic-stack pass, all indices with height at least $a_i$ are removed. The remaining top, if one exists, is the nearest earlier board with height strictly smaller than $a_i$. Hence, the valid interval begins immediately after it. The reverse pass finds the nearest strictly smaller board to the right in the same way. All boards between the boundaries have height at least $a_i$, while extending past either boundary encounters a smaller board. $\square$

**Lemma 2.** For a window beginning at $s$, the candidate generated by board $i$ has area $a_iw_i(s)$.

**Proof.** By Lemma 1, every board in $[l_i,r_i]$ has height at least $a_i$. Its intersection with the current window is contiguous, and every board in that intersection supports height $a_i$. Its width is its right endpoint minus its left endpoint plus one, which gives $w_i(s)$. Thus, the candidate rectangle is valid and has area $a_iw_i(s)$. $\square$

**Lemma 3.** The three inserted line segments evaluate to $a_iw_i(s)$ for every window start where $[l_i,r_i]$ intersects the window in a segment containing $i$.

**Proof.** Before the sliding window fully reaches the maximal interval, its right boundary moves right while the intersection's left boundary remains $l_i$. The width is $s+k-l_i$, which has slope $1$.

During the middle portion, either the full window lies inside $[l_i,r_i]$ or the full interval lies inside the window. The width is the constant $\min(k,r_i-l_i+1)$.

Afterward, the intersection's right boundary remains $r_i$ while its left boundary moves with $s$. The width is $r_i-s+1$, which has slope $-1$. Multiplying each expression by $a_i$ gives exactly the inserted lines. $\square$

**Lemma 4.** For every window, at least one generated candidate has area equal to its optimal rectangle.

**Proof.** Take an optimal rectangle spanning $[u,v]$, and choose an index $i\in[u,v]$ whose height is minimum on that range. The rectangle has height $a_i$, and every board in $[u,v]$ has height at least $a_i$. By Lemma 1,

$$
[u,v]\subseteq[l_i,r_i].
$$

Since $[u,v]$ also lies inside the current window, it is contained in the intersection used by candidate $i$. That candidate has the same height and at least the same width as the optimal rectangle. It is valid, so its area cannot exceed the optimum; therefore, its area equals the optimum. $\square$

**Theorem.** Each value output by the algorithm is the maximum advertisement area for its window.

**Proof.** By Lemmas 2 and 3, every inserted function represents a valid rectangle area wherever it applies. By Lemma 4, the candidates include an optimal rectangle for every window. The Li Chao query returns the maximum of all candidates valid at that start, so it returns exactly the optimal area. $\square$

## Implementation

The monotonic stacks remove heights using

```cpp
while (!s.empty() && a[s.back()] >= a[i]) s.pop_back();
```

The `>=` comparison leaves the nearest strictly smaller height on the stack. Equal-height boards can therefore belong to the same maximal interval.

The Li Chao tree stores a line

$$
f(x)=ax+b
$$

at each segment-tree node. A line is inserted only over the interval of window starts where its formula is valid. The range wrapper clips every segment to $[0,n-k]$.

All line coefficients, evaluations, and answers use `long long`. The largest area can reach

$$
2\cdot10^5\cdot10^9=2\cdot10^{14}.
$$

### Accepted C++17 implementation

```cpp
#include <bits/stdc++.h>
using namespace std;

using ll = long long;

struct Line {
    ll a = 0, b = 0;

    ll get(int x) {
        return a * x + b;
    }
};

int m;
vector<Line> tr;

void ins(Line z, int p, int l, int r) {
    int mid = (l + r) / 2;
    bool le = z.get(l) > tr[p].get(l);
    bool md = z.get(mid) > tr[p].get(mid);

    if (md) swap(z, tr[p]);
    if (l == r) return;

    if (le != md) ins(z, p * 2, l, mid);
    else ins(z, p * 2 + 1, mid + 1, r);
}

void add(Line z, int ql, int qr, int p, int l, int r) {
    if (qr < l || r < ql) return;

    if (ql <= l && r <= qr) {
        ins(z, p, l, r);
        return;
    }

    int mid = (l + r) / 2;
    add(z, ql, qr, p * 2, l, mid);
    add(z, ql, qr, p * 2 + 1, mid + 1, r);
}

void add(Line z, int l, int r) {
    l = max(l, 0);
    r = min(r, m - 1);
    if (l <= r) add(z, l, r, 1, 0, m - 1);
}

ll qry(int x, int p, int l, int r) {
    ll ans = tr[p].get(x);
    if (l == r) return ans;

    int mid = (l + r) / 2;
    if (x <= mid) ans = max(ans, qry(x, p * 2, l, mid));
    else ans = max(ans, qry(x, p * 2 + 1, mid + 1, r));

    return ans;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, k;
    cin >> n >> k;

    vector<int> a(n), l(n), r(n), s;
    for (int i = 0; i < n; i++) cin >> a[i];

    for (int i = 0; i < n; i++) {
        while (!s.empty() && a[s.back()] >= a[i]) s.pop_back();
        l[i] = s.empty() ? 0 : s.back() + 1;
        s.push_back(i);
    }

    s.clear();

    for (int i = n - 1; i >= 0; i--) {
        while (!s.empty() && a[s.back()] >= a[i]) s.pop_back();
        r[i] = s.empty() ? n - 1 : s.back() - 1;
        s.push_back(i);
    }

    m = n - k + 1;
    tr.resize(4 * m + 5);

    for (int i = 0; i < n; i++) {
        int x = l[i], y = r[i];
        int p = min(x, y - k + 1);
        int q = max(x, y - k + 1);
        ll h = a[i];

        add({h, h * (k - x)}, x - k + 1, p - 1);
        add({0, h * min(k, y - x + 1)}, p, q);
        add({-h, h * (y + 1)}, q + 1, y);
    }

    for (int i = 0; i < m; i++) {
        if (i) cout << ' ';
        cout << qry(i, 1, 0, m - 1);
    }

    cout << '\n';
}
```

## Complexity

The two monotonic-stack passes take $O(n)$ time. Each board creates at most three line segments. Inserting one line over an interval takes $O(\log^2 n)$, and each point query takes $O(\log n)$. Therefore, the total time complexity is $O(n\log^2 n)$.

The boundary arrays and Li Chao segment tree use $O(n)$ memory.

## Worked Example

For

$$
a=[4,1,5,3,3,2,4,1],
\qquad
k=3,
$$

consider the height $3$ at index $3$. Its maximal interval is $[2,4]$, corresponding to the boards $[5,3,3]$.

| Window start | Intersection width | Area at height $3$ |
|---:|---:|---:|
| $0$ | $1$ | $3$ |
| $1$ | $2$ | $6$ |
| $2$ | $3$ | $9$ |
| $3$ | $2$ | $6$ |
| $4$ | $1$ | $3$ |

The maximum over every board's contribution gives:

| Window | Best rectangle | Area |
|---|---|---:|
| $[4,1,5]$ | height $5$, width $1$ | $5$ |
| $[1,5,3]$ | height $3$, width $2$ | $6$ |
| $[5,3,3]$ | height $3$, width $3$ | $9$ |
| $[3,3,2]$ | height $2$, width $3$ | $6$ |
| $[3,2,4]$ | height $2$, width $3$ | $6$ |
| $[2,4,1]$ | height $4$, width $1$ | $4$ |

The output is

$$
5\quad6\quad9\quad6\quad6\quad4.
$$
