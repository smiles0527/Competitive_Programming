# Sliding Window Minimum

## Problem

- **Source:** [CSES 3221: Sliding Window Minimum](https://cses.fi/problemset/task/3221/)
- **Code:** [`View accepted C++ solution (minimum.cpp)`](../minimum.cpp)
- **Limits:** $1\le k\le n\le10^7$, $0\le x,a,b\le10^9$, and $1\le c\le10^9$.

The sequence is generated by

$$
x_1=x,
\qquad
x_i=(ax_{i-1}+b)\bmod c
$$

for $i=2,3,\ldots,n$.

For every length-$k$ window, compute its minimum. Output the bitwise XOR of all $n-k+1$ window minimums.

## Idea

Computing each window minimum independently takes $O(nk)$ time. With $n$ and $k$ as large as $10^7$, we need a data structure that processes each generated value in constant amortized time.

Unlike a sum, a minimum cannot be updated by simply deleting the outgoing value. If the deleted value was the minimum, we need to find the next minimum among the remaining values.

Store a deque of candidate indices such that their values are strictly increasing from front to back. For each new value $x_i$:

1. Remove the front index if it lies outside the new window.
2. Remove values from the back while they are at least $x_i$.
3. Insert $i$ at the back.

Suppose $x_j$ is deleted from the back when the newer value $x_i$ arrives. Then

$$
j<i
\qquad\text{and}\qquad
x_i\le x_j.
$$

Every future window containing $x_j$ also contains $x_i$ until $x_j$ expires. Since $x_i$ is no greater and expires later, $x_j$ can never again be needed as the minimum and may be deleted.

After these deletions, the candidate values are strictly increasing. Thus, the front of the deque is the minimum of the current window.

It is unnecessary to store the entire sequence. The implementation keeps the deque in circular arrays of length $k$.

## Algorithm

For each zero-based sequence index $i$ from $0$ to $n-1$:

1. Delete the front index if it is at most $i-k$, because it lies outside the current window.
2. While the deque is nonempty and its back value is at least `x`, delete the back.
3. Add the pair $(x,i)$ at the back.
4. If $i\ge k-1$, apply `ans ^= qv[head]`.
5. Advance the generator:

   $$
   x\gets(ax+b)\bmod c.
   $$

Output `ans`.

## Correctness

We prove that the algorithm outputs the XOR of all length-$k$ window minimums.

**Lemma 1.** Every index stored in the deque belongs to the current window.

**Proof.** Before processing index $i$, the algorithm deletes the front if its index is at most $i-k$. Such an index lies outside the current window $[i-k+1,i]$. Indices are added in increasing order, so every expired index must occur at the front. After expiration and insertion, all stored indices belong to the current window. $\square$

**Lemma 2.** The values stored in the deque are strictly increasing from front to back.

**Proof.** Before adding $x_i$, the algorithm deletes back elements while their values are at least $x_i$. After these deletions, either the deque is empty or its back value is strictly smaller than $x_i$. Adding $x_i$ therefore preserves strict increasing order. By induction, the deque values remain strictly increasing. $\square$

**Lemma 3.** Removing an element from the back cannot remove a value needed as a future window minimum.

**Proof.** Suppose $x_j$ is deleted when the newer value $x_i$ is added. Then $j<i$ and $x_i\le x_j$. Because $i$ is newer, it expires after $j$. Every future window containing $x_j$ also contains $x_i$, whose value is no larger. Thus, $x_j$ cannot be the unique minimum candidate in any future window and may be deleted. $\square$

**Lemma 4.** After processing index $i$, the deque front is the minimum of the current window.

**Proof.** By Lemma 1, every deque element belongs to the current window. By Lemma 3, removing dominated elements does not change the minimum. The deque therefore retains an element attaining the current minimum. By Lemma 2, its smallest value is at the front. $\square$

**Theorem.** The final value of `ans` is the XOR of all length-$k$ window minimums.

**Proof.** A complete window exists whenever $i\ge k-1$. For each such index, Lemma 4 shows that the deque front is exactly the minimum of the window ending at $i$. The endpoints $k-1,k,\ldots,n-1$ correspond to all $n-k+1$ windows, and each minimum is XORed into `ans` once. Therefore, `ans` is the required XOR. $\square$

## Implementation

The deque is represented by two circular arrays:

- `qv` stores candidate values;
- `qi` stores their sequence indices.

The variable `head` identifies the front, while `sz` is the number of stored elements. The logical back is at

$$
(\texttt{head}+\texttt{sz}-1)\bmod k.
$$

When an expired front is removed, `head` advances by one modulo $k$. Removing from the back only decreases `sz`.

Using `>=` in the back-deletion condition preserves the newest occurrence among equal values. The older equal value expires earlier and therefore cannot be more useful.

The generator parameters use `long long` because $ax+b$ may approach $10^{18}$. Individual generated values and the XOR answer fit in `int`, since every minimum is smaller than $c\le10^9$.

### Accepted C++17 implementation

```cpp
#include <bits/stdc++.h>
using namespace std;

using ll = long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, k;
    ll x, a, b, c;
    cin >> n >> k;
    cin >> x >> a >> b >> c;

    vector<int> qv(k), qi(k);
    int head = 0, sz = 0;
    int ans = 0;

    for (int i = 0; i < n; i++) {
        if (sz && qi[head] <= i - k) {
            head++;
            if (head == k) {
                head = 0;
            }
            sz--;
        }

        while (sz) {
            int p = head + sz - 1;
            if (p >= k) {
                p -= k;
            }

            if (qv[p] < x) {
                break;
            }

            sz--;
        }

        int p = head + sz;
        if (p >= k) {
            p -= k;
        }

        qv[p] = x;
        qi[p] = i;
        sz++;

        if (i >= k - 1) {
            ans ^= qv[head];
        }

        x = (a * x + b) % c;
    }

    cout << ans << '\n';
}
```

## Complexity

Each index is inserted once. It can be removed at most once, either from the front after expiring or from the back after being dominated. The total time complexity is therefore $O(n)$.

The circular deque stores at most $k$ values and indices, so the auxiliary space complexity is $O(k)$.

## Worked Example

For $n=8$, $k=5$, $x=3$, $a=7$, $b=1$, and $c=11$, the generated sequence is

$$
[3,0,1,8,2,4,7,6].
$$

| Index | Incoming | Candidate values after insertion | Window minimum | XOR answer |
|---:|---:|---|---:|---:|
| $0$ | $3$ | $[3]$ | - | - |
| $1$ | $0$ | $[0]$ | - | - |
| $2$ | $1$ | $[0,1]$ | - | - |
| $3$ | $8$ | $[0,1,8]$ | - | - |
| $4$ | $2$ | $[0,1,2]$ | $0$ | $0$ |
| $5$ | $4$ | $[0,1,2,4]$ | $0$ | $0$ |
| $6$ | $7$ | $[1,2,4,7]$ | $1$ | $1$ |
| $7$ | $6$ | $[2,4,6]$ | $2$ | $3$ |

The window minimums are $0,0,1,2$, whose XOR is $3$.
