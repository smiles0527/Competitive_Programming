# Sliding Window XOR

## Problem

- **Source:** [CSES 3426: Sliding Window XOR](https://cses.fi/problemset/task/3426/)
- **Code:** [`View accepted C++ solution (xor.cpp)`](./xor.cpp)
- **Limits:** $1\le k\le n\le10^7$, $0\le x,a,b\le10^9$, and $1\le c\le10^9$.

The sequence is generated by

$$
x_1=x,
\qquad
x_i=(ax_{i-1}+b)\bmod c
$$

for $i=2,3,\ldots,n$.

For every length-$k$ window, compute the XOR of its elements. Output the XOR of all $n-k+1$ window XORs.

## Conceptual Approach

A direct sliding-window approach can maintain the current XOR by applying XOR with the outgoing value:

$$
W_{l+1}=W_l\oplus x_l\oplus x_{l+k}.
$$

This runs in $O(n)$ time, but it requires storing the most recent $k$ values.

The final output is the XOR of all window XORs. Expand the complete expression and group equal elements from the sequence. Let

$$
W_l=x_l\oplus x_{l+1}\oplus\cdots\oplus x_{l+k-1}.
$$

In

$$
W_1\oplus W_2\oplus\cdots\oplus W_{n-k+1},
$$

each element $x_i$ appears exactly once for every window containing position $i$. Since

$$
v\oplus v=0,
$$

the element contributes to the answer only when the number of windows containing it is odd.

Using zero-based indices, a window beginning at $l$ contains position $i$ when

$$
l\le i\le l+k-1.
$$

Equivalently,

$$
i-k+1\le l\le i.
$$

A valid window start must also satisfy $0\le l\le n-k$. Therefore, the valid starts containing $i$ form the interval

$$
\max(0,i-k+1)\le l\le\min(i,n-k).
$$

The number of such windows is

$$
\operatorname{cnt}_i=min(i,n-k)-\max(0,i-k+1)+1.
$$

If $\operatorname{cnt}_i$ is odd, XOR $x_i$ into the answer. Otherwise, all appearances cancel. The sequence can be generated online without storing an array or circular buffer.

The generator variables require `long long` because $ax+b$ may approach $10^{18}$.

## Algorithm

For every zero-based index $i$ from $0$ to $n-1$:

1. Compute the first valid window start containing $i$:

   $$
   l=\max(0,i-k+1).
   $$

2. Compute the last valid window start containing $i$:

   $$
   r=\min(i,n-k).
   $$

3. The number of windows containing $i$ is $r-l+1$. If this number is odd, apply `ans ^= x`.
4. Generate the next sequence value.

Output `ans`.

## Correctness

We prove that the algorithm returns the XOR of all length-$k$ window XORs.

**Lemma 1.** For zero-based position $i$, the number of length-$k$ windows containing it is

$$
\operatorname{cnt}_i=min(i,n-k)-\max(0,i-k+1)+1.
$$

**Proof.** A window starting at $l$ contains $i$ exactly when

$$
l\le i\le l+k-1,
$$

which is equivalent to $i-k+1\le l\le i$. Valid window starts also satisfy $0\le l\le n-k$. Intersecting these ranges gives

$$
\max(0,i-k+1)\le l\le\min(i,n-k).
$$

The number of integers in this interval is the stated expression. $\square$

**Lemma 2.** The element $x_i$ contributes to the output if and only if $\operatorname{cnt}_i$ is odd.

**Proof.** When all window XORs are combined, $x_i$ appears once for every window containing position $i$. By Lemma 1, it appears $\operatorname{cnt}_i$ times. Since $v\oplus v=0$, pairs of occurrences cancel. An even number contributes zero, while an odd number leaves one copy of $x_i$. $\square$

**Theorem.** The final value of `ans` is the XOR of all window XORs.

**Proof.** The algorithm processes every position $i$. By Lemma 1, it computes exactly how many windows contain that position. By Lemma 2, it XORs $x_i$ into `ans` exactly when that element remains after all paired occurrences cancel. Therefore, `ans` contains precisely the contributions in the XOR of all window XORs. $\square$

## Implementation

The implementation does not calculate the individual window XORs. Instead, it determines whether each generated element appears an odd number of times in the resulting expression.

For every index,

```cpp
int l = max(0, i - k + 1);
int r = min(i, n - k);
```

describe the inclusive range of valid window starts containing that index. Because $k\le n$, every position belongs to at least one window, so $r-l+1\ge1$.

The generated value fits in `int`, but `x`, `a`, `b`, and `c` use `long long` to prevent overflow in

```cpp
x = (a * x + b) % c;
```

The current value is processed before the generator advances. Thus, the first processed value is $x_1$, which occupies zero-based index $0$.

### Accepted C++17 implementation

```cpp
#include <bits/stdc++.h>
using namespace std;

using ll = long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, k;
    ll x, a, b, c;
    cin >> n >> k;
    cin >> x >> a >> b >> c;

    int ans = 0;

    for (int i = 0; i < n; i++) {
        int l = max(0, i - k + 1);
        int r = min(i, n - k);

        if ((r - l + 1) & 1) {
            ans ^= x;
        }

        x = (a * x + b) % c;
    }

    cout << ans << '\n';
}
```

## Complexity

Each of the $n$ generated values is processed once with constant work. The time complexity is $O(n)$.

Only a constant number of variables is stored, so the auxiliary space complexity is $O(1)$.

## Worked Example

For $n=8$, $k=5$, $x=3$, $a=7$, $b=1$, and $c=11$, the generated sequence is

$$
[3,0,1,8,2,4,7,6].
$$

There are $n-k+1=4$ windows.

| Index $i$ | Value $x_i$ | First start $l$ | Last start $r$ | Containing windows | Contribution |
|---:|---:|---:|---:|---:|---:|
| $0$ | $3$ | $0$ | $0$ | $1$ | $3$ |
| $1$ | $0$ | $0$ | $1$ | $2$ | $0$ |
| $2$ | $1$ | $0$ | $2$ | $3$ | $1$ |
| $3$ | $8$ | $0$ | $3$ | $4$ | $0$ |
| $4$ | $2$ | $0$ | $3$ | $4$ | $0$ |
| $5$ | $4$ | $1$ | $3$ | $3$ | $4$ |
| $6$ | $7$ | $2$ | $3$ | $2$ | $0$ |
| $7$ | $6$ | $3$ | $3$ | $1$ | $6$ |

Only positions contained in an odd number of windows remain:

$$
3\oplus1\oplus4\oplus6=0.
$$

This matches the XOR of the individual window XORs:

$$
8\oplus15\oplus8\oplus15=0.
$$
